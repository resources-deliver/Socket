# Socket 多线程通信程序

## 简介

该文件夹下的 Client.cpp 和 Server.cpp 为多线程的 Socket 通信程序。Server.cpp 为服务器端，Client.cpp 为客户端。

### 特点
- 基于 TCP 协议的 Socket 通信
- 多线程实现，服务端可以同时连接多个客户端
- 每个客户端连接由独立线程处理，互不阻塞
- 支持客户端与服务器端的双向通信
- 实现了端口复用，服务端重启时不会出现端口占用问题

## 文件说明

- `Server.cpp` - 服务器端程序，负责监听和接受客户端连接，并为每个客户端创建独立线程进行通信
- `Client.cpp` - 客户端程序，负责连接服务器，并与服务器进行通信
- `README.md` - 说明文档

## 编译方法

服务端和客户端应该分开编译运行，分别使用以下命令：

```bash
# 编译服务端
g++ -std=c++11 -Wall -pthread Server.cpp -o server

# 编译客户端
g++ -std=c++11 -Wall Client.cpp -o client
```

**注意**：由于使用了 C++11 的多线程特性，编译时需要添加 `-pthread` 选项。

## 运行说明

### 启动服务端

```bash
./server
```

服务端默认配置：
- 监听端口：8080
- IP 地址：使用 INADDR_ANY（绑定到所有可用网卡）

### 启动客户端

```bash
./client
```

客户端默认配置：
- 连接端口：8080
- 服务器 IP：192.168.162.128（此处为本机IP）

**注意**：请根据实际网络环境修改 Client.cpp 中的服务器 IP 地址。

可以同时启动多个客户端连接到同一个服务器，服务器会为每个客户端创建独立的通信线程。

## 功能说明

### 服务端功能

1. 创建 Socket
2. 绑定 IP 地址和端口（包含端口复用设置）
3. 设置监听（最大同时连接数为 128）
4. 循环接受客户端连接
5. 显示每个客户端连接信息（IP 地址和端口）
6. 为每个客户端创建独立线程进行通信
7. 与客户端进行双向通信（接收消息并原样返回）
8. 处理客户端断开连接

### 客户端功能

1. 创建 Socket
2. 连接服务器
3. 与服务器进行双向通信
   - 每隔 1 秒发送一条消息
   - 接收并显示服务器返回的消息
4. 处理服务器断开连接

## 通信流程

1. 服务端启动并监听端口
2. 客户端连接服务器
3. 服务端接受连接，为该客户端创建独立线程
4. 建立连接后，客户端每隔 1 秒向服务器发送消息
5. 服务器接收消息后原样返回给客户端
6. 任一方断开连接，该客户端的通信线程结束
7. 服务端继续监听新的客户端连接

## 技术要点

### Socket 相关
- 使用 `socket()` 创建套接字
- 使用 `bind()` 绑定地址和端口
- 使用 `setsockopt()` 设置端口复用（SO_REUSEADDR）
- 使用 `listen()` 设置监听
- 使用 `accept()` 接受客户端连接
- 使用 `connect()` 连接服务器
- 使用 `read()` 和 `write()` 进行数据收发
- 使用 `inet_pton()` 和 `inet_ntop()` 进行 IP 地址转换
- 使用 `htons()` 和 `ntohs()` 进行端口字节序转换

### 多线程相关
- 使用 `std::thread` 创建线程
- 使用 `std::vector` 管理线程对象
- 使用 `emplace_back()` 向线程容器添加新线程
- 使用 `detach()` 分离线程，使其独立运行
- 每个客户端连接由独立线程处理，实现并发通信

## 注意事项

1. 该程序为多线程实现，可以同时处理多个客户端连接
2. 服务端使用了端口复用，即使异常退出也能快速重启
3. 请确保服务器 IP 地址和端口配置正确
4. 防火墙可能需要开放 8080 端口
5. 编译时需要添加 `-pthread` 选项以支持多线程
6. 多个客户端可以同时连接到服务器，每个客户端的通信互不影响
